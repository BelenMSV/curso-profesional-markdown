# Módulo 3: Sintaxis Extendida

¡Damos un paso más allá! 🚀

En este módulo abandonamos las especificaciones originales de Markdown para adentrarnos en la **Sintaxis Extendida** (conocida a menudo por sus siglas GFM, *GitHub Flavored Markdown*). 

A medida que Markdown se hizo popular, plataformas como GitHub se dieron cuenta de que los desarrolladores necesitaban más herramientas para documentar código y gestionar proyectos. Aprender a usar estas características adicionales marcará la diferencia entre un archivo `README.md` estándar y uno sobresaliente.

## 📚 Índice de contenidos

Haz clic en cada uno de los temas para acceder a su lección correspondiente:

1. [**Tablas**](./01-tablas/) - Organiza, estructura y compara datos de forma clara y visual.
2. [**Listas de Tareas**](./02-listas-de-tareas/) - Crea *checklists* interactivos, ideales para la gestión de proyectos o requisitos.
3. [**Notas al Pie**](./03-notas-al-pie/) - Añade referencias bibliográficas y aclaraciones sin romper el ritmo de lectura.
4. [**Tachado y Resaltado**](./04-tachado-y-resaltado/) - Marca correcciones temporales o subraya los conceptos más críticos de tu documento.
5. [**Alertas y Paneles**](./05-alertas-y-paneles/) - Destaca advertencias, consejos y notas con vistosos bloques de colores (soportados nativamente en GitHub).
