# Introducción a Markdown

¡Te damos la bienvenida al primer tema del curso! Antes de empezar a escribir código, es importante entender qué es exactamente Markdown y por qué se ha convertido en una **herramienta imprescindible para desarrolladores**, creadores de contenido y perfiles técnicos.

## ¿Qué es Markdown?

Markdown es un **lenguaje de marcado ligero** que permite dar formato a texto plano (legible en cualquier dispositivo) de forma sencilla y legible, **convirtiéndose fácilmente en HTML**. Su sintaxis es muy sencilla, intuitiva y potente, ya que está formada por caracteres simples fáciles de recordar (por ejemplo, `#`, `*`, `[]`, etc).

Para que veas un ejemplo rápido de cómo se escribe (usamos un bloque de texto plano para mostrarte la sintaxis sin que GitHub le aplique el formato visual):

```text
# Esto es un encabezado principal
**Esto es un texto en negrita**
* Esto es un elemento de una lista
```
## ¿Para qué se utiliza?

Es muy usado en **GitHub**, documentación técnica, blogs, foros, notas personales y últimamente para interactuar con sistemas de IA.

## Archivos y Entorno de Trabajo

Los documentos Markdown son simplemente archivos de **texto plano** con extensión `.md`. Puedes escribir Markdown desde cualquier editor de texto, aunque también existen herramientas especializadas y soporte dentro de cualquier editor de código (que cuentan con vistas previas para visualizar el resultado del texto formateado en Markdown).

## El estándar: CommonMark

En la actualidad, la sintaxis original ha evolucionado hacia el **estándar CommonMark**, una iniciativa para crear una especificación de Markdown completa. **No busca añadir muchas características** nuevas, sino **formalizar su uso**. Podemos decir que, hoy en día, CommonMark es el estándar real cuando nos referimos a Markdown básico.
